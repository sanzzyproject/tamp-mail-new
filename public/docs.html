TempMail Docs v1.2.0

Dokumentasi Sistem

Panduan referensi teknis lengkap mengenai arsitektur, keamanan, dan alur kerja sistem Temp Mail.

Ringkasan Eksekutif: Platform ini menyediakan layanan email sementara (disposable email) yang berfokus pada privasi pengguna dengan menggunakan arsitektur serverless dan penyimpanan data sisi-klien (client-side storage).

1. Arsitektur Sistem

Sistem dibangun di atas infrastruktur modern untuk menjamin kecepatan dan privasi data. Berikut adalah diagram alur data:

[User Device] <---> [Vercel Edge] <---> [Upstream]
     |                   |                  |
(IndexedDB)         (Node.js API)     (Email Server)
(HTML/JS UI)        (No Database)     (Scraper)

Stateless Backend

API Backend berjalan di Vercel Functions. Tidak ada database server yang menyimpan log email.

Local Persistence

Inbox disimpan menggunakan IndexedDB di browser HP Anda. Data aman di tangan Anda.

2. Spesifikasi & Keamanan

Untuk menjaga integritas dan privasi, aplikasi menerapkan standar ketat dalam penanganan data.

Komponen Teknologi Deskripsi Keamanan
Transport HTTPS / TLS Enkripsi end-to-end.
Storage IndexedDB Pesan disimpan lokal di perangkat.
Engine Cheerio.js Real-time processing tanpa cache server.
Catatan Privasi: Jangan gunakan email ini untuk layanan perbankan atau akun jangka panjang.

3. Referensi API Internal

Frontend berkomunikasi dengan backend melalui endpoints berikut:

Generate Email

GET /api?action=generate

Membuat identitas email baru. Mengembalikan objek JSON berisi alamat email.

Fetch Inbox

GET /api?action=inbox&email={user_email}

Mengambil daftar pesan terbaru secara real-time.

4. PWA (Mobile App)

Aplikasi mendukung instalasi native di Android. Service Worker menangani cache UI agar hemat kuota.

  • Strategy: Cache-First untuk UI.
  • Network-First: Untuk data email agar selalu update.
